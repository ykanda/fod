# bug-28 調査レポート

## 症状
- フィルタ文字列に日本語を入力すると、エントリリストのハイライト表示が欠ける。
- 例: `あああ.md` がある状態でフィルタ `あああ` を入力すると、3文字目の「あ」が半分欠けて表示される。

## 原因
- 画面幅計算や切り詰めが `len([]rune(...))` を使用しており、全角文字の表示幅を1として扱っている。
- 実際の描画は `lipgloss` が表示幅（全角2幅）を考慮するため、行幅の見積もりが過小になり、
  `Width(width)` で描画した際に末尾が切れてハイライトが欠ける。

## 修正方針
- 文字列の表示幅計算と切り詰めを `go-runewidth` ベースに統一する。
- `available` の算出、`truncateLine`/`truncateRunes` を表示幅基準に変更し、
  日本語などのマルチバイト文字を含む場合でもハイライト欠けを防ぐ。
